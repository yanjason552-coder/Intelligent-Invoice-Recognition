# 菜单结构修改测试

## 修改内容

### 1. 菜单名称变更
- 原：`销售订单查询`
- 新：`销售订单`

### 2. 新增子菜单
- `显示` - 查看销售订单详细信息
- `更改` - 修改销售订单信息  
- `清单` - 查看所有销售订单列表

### 3. 菜单层级结构
```
套料作业
├── 销售订单
│   ├── 显示
│   ├── 更改
│   └── 清单
├── 库存查询
├── 套料执行
├── 套料发布
└── 维护套料
```

## 测试步骤

1. 启动前端开发服务器
2. 打开浏览器访问应用
3. 检查左侧菜单栏
4. 点击"套料作业"展开菜单
5. 验证"销售订单"显示为文件夹图标
6. 点击"销售订单"下的子菜单项
7. 验证对应的TAB页是否正确打开

## 预期结果

- 菜单结构正确显示
- 子菜单可以正常点击
- TAB页标题正确显示
- 不同子菜单打开不同的页面内容
