# 销售订单查询功能说明

## 功能概述

新增了销售订单查询功能，以TAB页的形式在欢迎页面中显示。用户可以通过左侧菜单打开销售订单查询页面。

## 功能特点

### 1. TAB页管理
- 支持多个TAB页同时打开
- 可以关闭不需要的TAB页（除了欢迎页面）
- 自动切换到前一个TAB页当关闭当前TAB时
- 防止重复打开相同的TAB页

### 2. 销售订单查询
- **搜索功能**：支持按订单号或客户名称搜索
- **状态筛选**：支持按订单状态筛选（已确认、生产中、已完成、已取消）
- **数据展示**：表格形式展示订单信息
- **状态标识**：不同状态用不同颜色标识
- **操作按钮**：每个订单都有"查看详情"按钮

### 3. 响应式设计
- 移动端使用抽屉式菜单
- 桌面端使用固定侧边栏
- TAB页适配不同屏幕尺寸

## 使用方法

### 1. 打开销售订单查询
1. 登录系统后，在左侧菜单中找到"套料作业"分组
2. 点击"销售订单查询"菜单项
3. 系统会自动在欢迎页面打开一个新的TAB页

### 2. 搜索订单
1. 在搜索表单中输入订单号或客户名称
2. 选择订单状态（可选）
3. 点击"搜索"按钮
4. 系统会显示匹配的订单记录

### 3. 重置搜索
- 点击"重置"按钮可以清空搜索条件并显示所有订单

### 4. 关闭TAB页
- 点击TAB页上的"×"按钮可以关闭该TAB页
- 欢迎页面TAB不可关闭

## 技术实现

### 文件结构
```
frontend/src/
├── routes/_layout/index.tsx          # 主页面（包含TAB页管理）
├── components/Common/SidebarItems.tsx # 侧边栏菜单
└── components/Items/SalesOrderQuery.tsx # 销售订单查询组件
```

### 核心功能
1. **事件通信**：使用自定义事件在菜单和主页面之间通信
2. **状态管理**：使用React useState管理TAB页状态
3. **组件复用**：SalesOrderQuery组件可以独立使用
4. **数据模拟**：使用模拟数据展示功能

### 数据字段
- 订单号（id）
- 客户名称（customerName）
- 订单日期（orderDate）
- 订单状态（status）
- 订单金额（totalAmount）
- 商品数量（items）
