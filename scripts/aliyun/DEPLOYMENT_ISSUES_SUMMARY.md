## 1. Docker镜像源配置问题 ⚠️

### 问题描述
- **时间**: 部署初期
- **症状**: Docker无法拉取镜像，出现DNS解析失败错误
- **错误信息**: 
  ```
  failed to resolve source metadata for docker.io/library/node:20: 
  dial tcp: lookup docker.mirrors.ustc.edu.cn on 127.0.0.53:53: no such host
  ```

### 根本原因
- DNS服务器无法解析镜像源域名
- 中科大镜像源（docker.mirrors.ustc.edu.cn）可能暂时不可用
- Docker镜像源配置中的DNS解析失败

### 解决方案
1. **修复DNS配置**：更新 `/etc/resolv.conf`，添加公共DNS服务器
2. **更换镜像源**：使用其他可用的镜像源（DaoCloud、Docker Proxy等）
3. **使用阿里云镜像加速器**（推荐）：如果有阿里云账号，使用专属加速地址
4. **临时禁用镜像源**：直接使用Docker Hub（如果网络允许）

### 状态
✅ **已解决** - 通过更新镜像源配置和DNS设置解决

## 2. 磁盘空间不足问题 🔴

### 问题描述
- **时间**: 部署过程中
- **症状**: 
  - 磁盘使用率达到92%（20G磁盘，已使用17G，仅剩1.5G）
  - 无法创建新文件：`No space left on device`
  - Docker构建失败

### 根本原因
- 服务器磁盘容量较小（仅20G）
- Docker镜像、容器、日志占用大量空间
- 未及时清理未使用的Docker资源
- 系统日志文件积累

### 影响范围
- 无法继续部署
- 无法创建新文件
- Docker操作可能失败

### 解决方案
1. **清理Docker未使用资源**：
   ```bash
   docker system prune -af --volumes
   ```

2. **清理Docker日志**：
   ```bash
   find /var/lib/docker/containers -name "*.log" -type f -delete
   ```

3. **清理系统日志**：
   ```bash
   journalctl --vacuum-time=7d
   ```

4. **定期清理策略**：
   - 设置定时任务清理Docker资源
   - 限制Docker日志文件大小
   - 定期清理系统日志

### 优化建议
- **增加磁盘容量**：考虑升级到更大的磁盘（40G或更大）
- **使用数据盘**：将Docker数据目录迁移到独立数据盘
- **配置日志轮转**：限制日志文件大小和保留时间
- **定期维护**：建立定期清理机制

### 状态
⚠️ **部分解决** - 已清理空间，但需要长期优化策略

## 3. 数据库连接超时问题 ⚠️

### 问题描述
- **时间**: 服务启动时
- **症状**: 
  - 后端服务启动时出现连接超时警告
  - 错误信息：`connection timeout expired`
  - 服务可以启动，但数据库连接测试失败

### 错误信息
```
WARNING:app.core.db:数据库连接测试失败: 
(psycopg.errors.ConnectionTimeout) connection timeout expired
INFO:app.core.db:引擎已创建，将在实际使用时自动重连
```

### 根本原因
- 数据库服务器（8.145.33.61:50511）可能暂时不可达
- 网络延迟或防火墙阻止
- 连接超时设置（10秒）可能过短
- 数据库服务器负载过高

### 影响范围
- **不影响服务启动**：服务可以正常启动
- **有自动重连机制**：配置了 `pool_pre_ping=True`
- **实际使用时可能正常**：首次连接失败，实际使用时会自动重试

### 解决方案
1. **检查网络连通性**：
   ```bash
   ping 8.145.33.61
   timeout 5 bash -c "echo > /dev/tcp/8.145.33.61/50511"
   ```

2. **检查防火墙规则**：
   - 确保数据库服务器端口50511已开放
   - 确保应用服务器可以访问该端口

3. **使用psql测试连接**：
   ```bash
   psql -h 8.145.33.61 -p 50511 -U postgres -d invoice_db
   ```

4. **调整连接超时时间**（如果需要）：
   - 修改 `connect_timeout` 参数
   - 增加连接池超时时间

### 状态
⚠️ **待确认** - 需要进一步诊断，可能不影响实际使用

## 4. 部署脚本文件缺失问题 ⚠️

### 问题描述
- **时间**: 首次部署尝试
- **症状**: 
  - 脚本文件不存在：`scripts/aliyun/prepare-deployment.sh: No such file or directory`
  - 项目文件未上传到服务器

### 根本原因
- 项目代码未完整上传到服务器
- 部署脚本在本地创建但未同步到服务器
- Git仓库可能未克隆或文件未复制

### 解决方案
1. **在服务器上直接创建脚本**：使用heredoc方式创建脚本文件
2. **上传项目文件**：使用Git克隆或SCP上传
3. **创建脚本创建工具**：提供一键创建所有部署脚本的工具

### 状态
✅ **已解决** - 通过直接在服务器上创建脚本解决

## 6. 环境变量配置不完整问题 ⚠️

### 问题描述
- **时间**: 部署准备阶段
- **症状**: 
  - `.env`文件缺失或配置不完整
  - 必需的环境变量未设置
  - 使用默认值（如changethis）

### 根本原因
- 未创建生产环境配置文件
- 安全密钥未生成
- 数据库配置未正确设置

### 解决方案
1. **创建环境变量模板**：`.env.production.template`
2. **自动生成安全密钥**：在准备脚本中自动生成SECRET_KEY和POSTGRES_PASSWORD
3. **部署前检查**：创建检查脚本验证所有必需配置

### 状态
✅ **已解决** - 通过创建模板和自动生成脚本解决
