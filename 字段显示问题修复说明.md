# 字段显示问题修复说明

## 问题诊断

### 问题现象
发票详情页面中，`normalized_fields` 字段未正常显示。

### 根本原因
从后端日志分析发现：
1. **数据丢失**：`normalized_fields` 在处理过程中丢失，最终只剩下 `_debug_info`
2. **日志证据**：
   ```
   normalized_fields 字段数: 1
   normalized_fields 键: ['_debug_info']
   debug_normalized_fields_len: 0
   debug_normalized_fields_keys: []
   ```

### 问题位置
`backend/app/api/routes/invoice.py` 第2478-2496行：
- 在添加调试信息时，`normalized_fields` 已经是空字典
- 添加 `_debug_info` 后，`normalized_fields` 只包含调试信息，没有实际数据

## 修复方案

### 修复1: 增强数据检查（第2478-2520行）
在添加调试信息前，检查 `normalized_fields` 是否为空：
- 如果为空，尝试直接从数据库重新查询
- 确保数据完整性后再添加调试信息

### 修复2: 创建响应对象前检查（第2455-2500行）
在创建响应对象前：
- 检查 `normalized_fields` 是否只有 `_debug_info`
- 如果是，重新从数据库查询并恢复数据
- 确保响应对象包含完整的字段数据

### 修复3: 增强日志记录（第2174-2208行）
添加更详细的日志：
- 记录 `normalized_fields_db` 的完整内容
- 记录数据处理过程中的每个步骤
- 便于后续问题排查

## 修复后的流程

1. **查询识别结果**：从数据库查询 `normalized_fields`
2. **数据处理**：解析 JSONB 字段，转换为字典
3. **数据验证**：检查数据是否完整
4. **数据恢复**：如果数据丢失，重新从数据库查询
5. **添加调试信息**：只在数据完整时添加 `_debug_info`
6. **创建响应对象**：确保包含完整的 `normalized_fields`

## 测试建议

1. **重启后端服务**：使修复生效
2. **访问发票详情页面**：检查字段是否正常显示
3. **查看后端日志**：确认数据是否正确查询和处理
4. **检查前端控制台**：确认 API 返回的数据格式

## 预期结果

修复后，`normalized_fields` 应该包含：
- `date`: 日期
- `part_no`: 零件号
- `doc_type`: 文档类型
- `part_name`: 零件名称
- `drawing_no`: 图号
- `form_title`: 表单标题
- `inspector_name`: 检验员
- `overall_result`: 总体结果
- `items`: 检验项目列表（13项）
- `_debug_info`: 调试信息（可选）

前端应该能正常显示这些字段。

## 注意事项

1. **调试信息**：`_debug_info` 不应该影响字段显示，前端代码会自动过滤
2. **数据完整性**：如果数据库中的 `normalized_fields` 本身就是空的，需要检查识别流程
3. **日志监控**：关注后端日志中的错误和警告信息
